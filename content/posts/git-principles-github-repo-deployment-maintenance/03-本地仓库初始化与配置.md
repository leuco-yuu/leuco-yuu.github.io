# 本地仓库初始化与配置

## 初始化身份配置

Git 提交会记录作者和提交者信息。首次使用 Git 时，应配置用户名和邮箱：

```bash
git config --global user.name "Alice Zhang"
git config --global user.email "alice@example.com"
```

配置级别包括 system、global、local。优先级从低到高为 system<global<local。项目级配置写入 `.git/config`，适合针对某个仓库使用不同邮箱或远程策略。

查看配置来源：

```bash
git config --list --show-origin
```

常用配置示例：

```bash
git config --global init.defaultBranch main
git config --global pull.rebase false
git config --global fetch.prune true
git config --global core.autocrlf input
git config --global core.editor "code --wait"
```

| 配置 | 作用 |
| --- | --- |
| `init.defaultBranch` | 新仓库默认分支名 |
| `pull.rebase` | `git pull` 默认采用 merge 还是 rebase |
| `fetch.prune` | fetch 时清理远程已删除分支的本地缓存 |
| `core.autocrlf` | 控制换行符转换策略 |
| `core.editor` | 指定提交说明编辑器 |

## 初始化普通仓库

在项目目录中初始化 Git 仓库：

```bash
mkdir demo-service
cd demo-service
git init -b main
```

`git init` 会创建 `.git` 目录，并生成对象、引用、模板等内部结构。常用参数如下：

| 参数 | 说明 | 示例 |
| --- | --- | --- |
| `-b <name>` / `--initial-branch=<name>` | 指定初始分支名 | `git init -b main` |
| `--bare` | 创建裸仓库，通常用于服务器端 | `git init --bare repo.git` |
| `--template=<dir>` | 使用模板目录 | `git init --template=~/git-template` |
| `--separate-git-dir <dir>` | 将 `.git` 数据存放到其他位置 | `git init --separate-git-dir /data/git/demo.git` |
| `--object-format=<format>` | 指定对象哈希格式 | `git init --object-format=sha1` |
| `--shared[=<permissions>]` | 创建共享仓库权限模式 | `git init --shared=group` |

对普通项目，推荐使用 `git init -b main`。对服务器或内部共享仓库，可以考虑裸仓库，但现代团队通常直接使用 GitHub、Gitee、GitLab 等平台，不再手动维护裸仓库。

> [!ANNOT]
>
> 裸仓库只保存 Git 仓库数据，不检出工作区文件。

## 克隆已有仓库

克隆远程仓库：

```bash
git clone https://github.com/octo-org/demo-service.git
```

常用参数如下：

| 参数 | 说明 | 示例 |
| --- | --- | --- |
| `--branch <name>` / `-b <name>` | 克隆后检出指定分支或标签 | `git clone -b develop <url>` |
| `--depth <n>` | 浅克隆最近 n 层历史 | `git clone --depth 1 <url>` |
| `--single-branch` | 只克隆单个分支历史 | `git clone --single-branch -b main <url>` |
| `--recurse-submodules` | 同时初始化并克隆子模块 | `git clone --recurse-submodules <url>` |
| `--origin <name>` | 指定默认远程名，默认是 origin | `git clone --origin upstream <url>` |
| `--bare` | 克隆为裸仓库 | `git clone --bare <url>` |
| `--mirror` | 镜像克隆所有引用和远程配置 | `git clone --mirror <url>` |

浅克隆适合 CI 快速拉取或只需最新代码的场景，但会限制完整历史分析、某些变基操作和标签推导。长期开发本地仓库建议保留完整历史。

## 忽略文件与属性文件

`.gitignore` 用于声明不纳入版本控制的路径。典型规则如下：

```gitignore
node_modules/
dist/
.env
*.log
.DS_Store
```

| 写法 | 含义 |
| --- | --- |
| `dist/` | 忽略目录 |
| `*.log` | 忽略所有 `.log` 文件 |
| `!keep.log` | 例外规则，取消忽略 |
| `/build` | 只匹配仓库根目录下的 build |
| `**/tmp` | 匹配任意层级的 tmp |

已经被 Git 跟踪的文件不会因为新增 `.gitignore` 自动停止跟踪。需要执行：

```bash
git rm --cached .env
```

`.gitattributes` 用于定义路径属性，如换行符、语言识别、合并驱动、归档行为。示例：

```gitattributes
* text=auto
*.sh text eol=lf
*.bat text eol=crlf
*.png binary
*.jpg binary
```

## 提交前的基础文件

一个可维护仓库通常至少包含以下文件：

| 文件 | 作用 |
| --- | --- |
| `README.md` | 项目介绍、安装、运行、使用说明 |
| `LICENSE` | 开源许可或内部版权声明 |
| `.gitignore` | 忽略构建产物、依赖目录、密钥文件 |
| `.gitattributes` | 统一换行符与文件属性 |
| `CONTRIBUTING.md` | 贡献流程、分支规范、提交规范 |
| `CHANGELOG.md` | 版本变更记录 |
| `CODEOWNERS` | 平台代码所有者规则 |

示例初始化：

```bash
cat > README.md <<'EOF'
Demo Service

A minimal service used to demonstrate Git workflow.
EOF

cat > .gitignore <<'EOF'
.env
node_modules/
dist/
*.log
EOF

git add README.md .gitignore
git commit -m "chore: initialize repository"
```

## SSH 与 HTTPS 访问方式

远程仓库常见访问方式有 HTTPS 和 SSH。HTTPS 配置简单，适合临时环境和新手；SSH 适合长期开发机，认证体验更稳定。

生成 SSH key：

```bash
ssh-keygen -t ed25519 -C "alice@example.com"
```

查看公钥：

```bash
cat ~/.ssh/id_ed25519.pub
```

把公钥添加到 GitHub 或 Gitee 后，测试连接：

```bash
ssh -T git@github.com
ssh -T git@gitee.com
```

在多账号环境中，可以使用 `~/.ssh/config` 区分主机别名：

```sshconfig
Host github-work
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_work

Host gitee-personal
  HostName gitee.com
  User git
  IdentityFile ~/.ssh/id_ed25519_personal
```

远程地址可写为：

```bash
git remote add origin git@github-work:org/demo-service.git
```
