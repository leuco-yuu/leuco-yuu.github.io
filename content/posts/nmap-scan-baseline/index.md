Nmap 的命令很多，但实验记录真正需要的是稳定的基线。基线不是最激进的扫描参数，而是一组能反复使用、便于比较、不会让结果过度失真的步骤。

本文只讨论授权实验环境中的扫描记录，不面向任何未授权目标。

# 主机发现

如果目标网段确定，可以先做轻量发现：

```powershell
nmap -sn 192.168.56.0/24
```

这一步的重点是建立资产候选列表，而不是证明主机一定不存在。实验网络里防火墙、虚拟网卡和桥接模式都会影响发现结果，所以发现结果需要和拓扑记录一起看。

# 端口探测

常用基线可以从 TCP 常见端口开始：

```powershell
nmap -sS -T3 --top-ports 1000 -oA scans/top-ports 192.168.56.10
```

`-T3` 不追求速度，优先保持结果稳定。`-oA` 同时保存普通文本、grepable 和 XML，后面做 diff 或导入工具会方便一些。

# 服务识别

端口开放之后再做服务版本识别：

```powershell
nmap -sV -sC -p 22,80,445 -oA scans/service 192.168.56.10
```

我通常不会一开始就对全端口挂上所有脚本。先缩小范围，再针对开放服务补充验证，记录更清楚，也更容易定位误报。

# 记录格式

每次扫描至少记四件事：

- 授权范围和目标地址
- 执行时间和网络位置
- 原始命令
- 输出文件路径和结论摘要

这些信息比单独贴一个端口列表更有价值，因为扫描结果总是和当时的网络条件绑定在一起。

# 小结

好的扫描流程不是“跑更多参数”，而是让每一步都有目的：发现、探测、识别、验证。基线稳定之后，再根据目标服务逐步加深，结果才更容易复盘。
